# 方案

## 总体流程

项目保留现有主流程：

1. `open_58_with_cdp()` 负责查找 Edge、准备独立资料目录、读取运行时长，并通过 CDP 获取浏览器、上下文和页面。
2. `run_periodically()` 负责按轮次循环执行，每轮结束后等待 `REFRESH_INTERVAL_SECONDS`，当前值为 600 秒。
3. `run_once()` 负责单轮业务流程。
4. `click_matching_online_chat()` 负责候选人筛选和在线沟通点击。

## 页面准备

新增 `is_target_page(page)`，统一判断当前页面是否处于目标页。

新增 `ensure_target_page(context, page, login_timeout_seconds)`，集中处理单轮开始前的页面准备：

1. 当前已经在目标页时，执行 `page.reload(wait_until="domcontentloaded")`。
2. 当前不在目标页时，执行 `page.goto(TARGET_URL, wait_until="domcontentloaded")`。
3. 如果进入登录页，复用 `wait_for_login()` 等待用户手动登录。
4. 登录后如果仍不在目标页，再执行 `page.goto(TARGET_URL, wait_until="domcontentloaded")`。
5. 返回最终用于后续业务流程的 `page`。

## 单轮执行

`run_once()` 只保留单轮业务编排：

1. 调用 `ensure_target_page()` 准备目标页。
2. 等待 `domcontentloaded`。
3. 调用 `wait_for_candidate_list(page)` 等待候选人列表。
4. 调用 `page.bring_to_front()` 把页面置前。
5. 调用 `click_matching_online_chat(page, cycle)` 执行筛选和点击。
6. 返回当前页面对象，供下一轮继续使用。

## 周期刷新

`run_periodically()` 不改变原有循环结构：

1. 每轮调用 `run_once()`。
2. 浏览器连接断开时，继续使用现有重连逻辑。
3. 本轮完成或失败后，按运行时长计算等待时间。
4. 等待结束后进入下一轮。

由于 `run_once()` 的页面准备阶段已经区分目标页和非目标页，下一轮开始时如果页面仍在目标页，会执行真实刷新；如果页面不在目标页，会先进入目标页，再继续完整单轮流程。

## 剩余时间补齐

新增 `round_up_to_refresh_interval(seconds)`，将有限运行时长下的剩余秒数按 `REFRESH_INTERVAL_SECONDS` 向上补齐。

`run_periodically()` 在有限运行时长下使用补齐后的剩余时间：

1. 单轮执行时传入补齐后的剩余时间作为登录等待上限。
2. 本轮结束后，如果原始剩余时间大于 0，则等待时间固定为 10 分钟。
3. 日志中的剩余运行时间使用补齐后的整 10 分钟值。
4. 如果等待 10 分钟会超过原始截止时间，记录一次性标记，允许等待后的下一轮继续执行。
5. 这次补齐后的下一轮执行完成后，如果已经超过原始截止时间，则正常退出，避免无限延长。

## 点击诊断

新增 `format_candidate_log_label(candidate, target_key)`，统一输出候选人姓名、年龄和本轮去重 key。

`click_matching_online_chat()` 在以下位置输出诊断日志：

1. 命中列表生成后，打印每个命中候选人的序号、姓名、年龄和 key。
2. 每次进入命中候选处理时，打印当前处理对象。
3. 因本轮已处理过相同 key 被跳过时，打印跳过原因。
4. 因等待恢复断点被跳过时，打印跳过原因和当前断点。
5. 页面未找到对应行或按钮时，打印跳过原因。
6. 点击前发现按钮被遮挡时，打印遮挡诊断。

## CDP 端口识别

新增 `parse_running_edge_cdp_port(command_lines, user_data_dir, port_is_ready)`，将 Edge 命令行解析逻辑从进程读取中拆出，支持中文路径和带引号的 `--user-data-dir`。

`read_running_edge_cdp_port()` 读取 PowerShell 输出时设置 UTF-8：

1. PowerShell 内设置 `[Console]::OutputEncoding` 和 `$OutputEncoding` 为 UTF-8。
2. Python `subprocess.run()` 使用 `encoding="utf-8"` 和 `errors="replace"`。
3. 解析命令行时不再直接用字符串包含判断，而是用正则提取 `--user-data-dir`、`--profile-directory` 和 `--remote-debugging-port`。
4. 提取端口后继续访问 `/json/version`，确认端口真实可用后再复用。

## 候选人 key

`infoid` 在当前页面中会被多个不同候选人共享，因此不再参与本轮去重、断点 key 和页面行优先定位。

候选人 key 生成规则调整为：

1. 优先使用 `resumeid`。
2. 没有 `resumeid` 时使用 `name + age`。

候选人数据归一化时，`infoid` 和 `resumeid` 分开保存，不再把 `resumeId` 塞入 `infoid` 字段。

页面行重新定位时，优先使用 `resumeid`，其次使用姓名和年龄；不再优先使用 `infoid`。

页面快照中不再通过 `outerHTML` 正则推断 `infoid`，避免误抓同一页其他候选人的 id。

## 保持不变

本次不调整以下逻辑：

1. 目标地址 `TARGET_URL`。
2. 10 分钟间隔 `REFRESH_INTERVAL_SECONDS`。
3. 登录等待策略。
4. Edge 启动和 CDP 重连逻辑。
5. 候选人接口请求参数。
6. 年龄和性别筛选规则。
7. 在线沟通按钮识别和点击规则。

## 到时自动关闭开关

本次采用单一方案：在现有运行时长菜单中加入可记忆开关，不增加额外二次确认。

### 配置持久化

新增独立配置文件 `app_config.json`，只保存：

1. `auto_close_edge_on_exit`：是否在到达设定运行时间后自动关闭当前 `edge_profile` 对应的 Edge。

运行中的断点状态仍使用 `runtime_state.json`，不与开关配置混用。

### 启动菜单

`prompt_run_duration_seconds()` 调整为：

1. 保留 `1~8` 小时选项不变。
2. `0` 表示永久执行。
3. `9` 表示“到时关闭浏览器开关”，并显示当前是“关浏览器”还是“不关浏览器”。
4. `9` 被选择后，立即切换状态并写入 `app_config.json`，然后重新显示菜单。
5. `10` 表示“测试运行 1 分钟”，只影响本次运行时长为 60 秒。
6. 用户输入 `1~8`、`0` 或 `10` 时，按当前已保存的开关状态启动。

### 到时退出行为

`run_periodically()` 新增开关参数：

1. 运行时间到达后始终正常退出程序。
2. 当 `auto_close_edge_on_exit` 为开时，先按当前 `user_data_dir + profile-directory` 筛选并关闭对应 `msedge.exe` 进程，再退出程序。
3. 当 `auto_close_edge_on_exit` 为关时，保持现状，不主动关闭 Edge。

### 关闭范围

关闭逻辑只处理当前项目 `edge_profile` 对应的 Edge：

1. 通过 PowerShell 读取 `msedge.exe` 进程命令行。
2. 匹配 `--user-data-dir=<当前 edge_profile>`。
3. 匹配 `--profile-directory=Default`。
4. 对命中的进程执行关闭。

由于程序本身支持接管已存在的同数据目录 Edge，这种按数据目录关闭的方式比只依赖本次 `Popen` 句柄更完整。

## 远程禁用校验

本次采用单一方案：仅在 `exe` 启动时做一次 TCP 握手校验，只有收到预期禁用响应才阻止启动。

### 客户端校验

新增远程禁用检查函数，放在 `main()` 进入 `open_58_with_cdp()` 之前执行：

1. 仅当 `sys.frozen` 为真时执行。
2. 使用短超时连接 `144.34.232.235:39000`。
3. 连接成功后发送 `PING_58\n`。
4. 读取响应并去掉空白。
5. 响应精确等于 `DISABLE_58` 时，提示“功能已禁用”，并中止启动。
6. 连接失败、超时、拒绝连接、响应为空、响应不匹配，都直接放行。

### 启动流程

`main()` 调整为：

1. 保留 `ensure_local_shortcut()`。
2. 新增远程禁用校验。
3. 仅当校验未命中禁用时，才进入 `open_58_with_cdp()`。

这样禁用命中时不会出现运行时长菜单，也不会启动浏览器。

### 服务端脚本

新增 `server/disable_switch_server.py`：

1. 使用 Python 标准库 `socketserver` 监听 `0.0.0.0:39000`。
2. 收到 `PING_58` 时返回 `DISABLE_58\n`。
3. 其他请求不返回禁用标志。
4. 脚本运行中表示“禁用开启”；脚本停止表示“禁用关闭”。

### 使用方式

1. 服务器上执行 `uv run server/disable_switch_server.py` 开启禁用。
2. 停止该脚本即可恢复允许启动。
3. 仅双击 `exe` 会触发校验，脚本运行方式不受影响。
